# Requisitos do Sistema – CUIDAR

## Visão Geral

O sistema CUIDAR tem como objetivo gerenciar informações de residentes de uma ILPI, garantindo organização, segurança e rastreabilidade dos dados.

---

# Requisitos Funcionais

## RF01 – Cadastro de Residentes
O sistema deve permitir cadastrar um residente com seus dados pessoais.

## RF02 – Edição de Residentes
O sistema deve permitir editar informações do residente.

## RF03 – Inativação de Residentes
O sistema deve permitir inativar residentes sem excluir seus dados.

## RF04 – Listagem de Residentes
O sistema deve exibir uma lista de residentes cadastrados.

## RF05 – Busca de Residentes
O sistema deve permitir buscar residentes por nome ou CPF.

## RF06 – Visualização Detalhada
O sistema deve permitir visualizar todos os dados de um residente.

---

## RF07 – Criação de Prontuário
O sistema deve criar um prontuário para cada residente.

## RF08 – Registro de Informações Médicas
O sistema deve permitir registrar:
- doenças crônicas
- alergias
- tipo sanguíneo

---

## RF09 – Registro Clínico
O sistema deve permitir registrar evoluções e observações clínicas.

## RF10 – Histórico Clínico
O sistema deve armazenar histórico cronológico de registros clínicos.

---

## RF11 – Cadastro de Medicamentos
O sistema deve permitir cadastrar medicamentos vinculados a um residente.

## RF12 – Gerenciamento de Medicamentos
O sistema deve permitir definir:
- dosagem
- frequência
- horários

---

## RF13 – Registro de Rotinas
O sistema deve permitir registrar atividades diárias:
- alimentação
- banho
- higiene

---

## RF14 – Cadastro de Funcionários
O sistema deve permitir cadastrar funcionários.

## RF15 – Controle de Acesso
O sistema deve diferenciar tipos de usuário:
- Administrador
- Enfermeiro
- Cuidador

---

## RF16 – Visualização de Relatórios
O sistema deve permitir visualizar:
- residentes
- prontuários
- medicamentos
- registros clínicos

---

# Requisitos Não Funcionais

## RNF01 – Segurança
O sistema deve garantir controle de acesso por tipo de usuário.

## RNF02 – Integridade de Dados
O sistema não deve permitir duplicidade de CPF.

## RNF03 – Usabilidade
O sistema deve possuir interface simples e intuitiva.

## RNF04 – Desempenho
O sistema deve responder rapidamente às ações do usuário.

## RNF05 – Persistência
Os dados devem ser armazenados de forma permanente.

## RNF06 – Rastreabilidade
O sistema deve registrar ações importantes realizadas.

---

# Observação Final

Os requisitos aqui definidos são base para o desenvolvimento do sistema, garantindo que todas as funcionalidades atendam às necessidades da instituição e dos usuários.
